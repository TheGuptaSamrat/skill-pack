# Partitioning Core Rules

- treat partitioning as a volume-management and operational decision, not a default modeling trick
- measure current volume, growth, and filter pattern before proposing partitioning
- align partition decisions with Basis and HANA administration, not only application development
- pair partitioning discussions with archiving, residence-time, and result persistence considerations
- document the consuming filters and runtime pain points before recommending a partition key
- do not propose exact partition layouts without concrete table and workload evidence
